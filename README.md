# 사용자 맞춤형 스마트 햄버거 키오스크 시스템

---

실시간 메뉴 선택과 장바구니 관리, 모의 결제 프로세스를 통합하여 실제 매장 환경을 구현한 C# WinForms 기반 주문 관리 시스템입니다.

# 프로젝트 목표

본 프로젝트는 실제 프랜차이즈 키오스크의 사용자 경험(UX)을 분석하여 **직관적인 주문 흐름**과 **데이터 무결성**을 보장하는 관리 시스템 구축을 목표로 합니다.

C# .NET 프레임워크를 사용하여 다음과 같은 핵심 기능을 제공합니다.

- **카테고리별 메뉴 관리:** 버거, 음료, 사이드 메뉴를 별도 폼으로 분리하여 객체 지향적 설계 구현
- **실시간 장바구니 시스템:** 메뉴 선택 시 메인 화면 리스트박스에 즉시 반영 및 개별 삭제 기능 제공
- **동적 가격 산출:** Regex 및 정수 파싱을 활용해 메뉴 추가/삭제 시 총 금액 자동 업데이트
- **결제 수단 모의 구현:** 카드, 은행, 포인트 등 콤보박스 연동을 통한 단계별 결제 정보 선택
- **예외 처리:** 장바구니 빈 상태 결제 방지 및 사용자 최종 확인 다이얼로그 시스템 구축

---

# 기대효과

- **사용자 편의성:** 카테고리별 UI 분리를 통해 주문 복잡도를 낮추고 직관적인 결제 프로세스 경험을 제공합니다.
- **데이터 관리 효율:** 메인 폼과 서브 폼 간의 `Owner` 속성 활용 및 메서드 호출로 효율적인 데이터 전송 구조를 확립합니다.
- **시스템 확장성:** 사용자 정의 컨트롤(UserControl)을 활용하여 향후 새로운 메뉴나 기능 추가 시 유지보수가 용이합니다.

## 🛠 사용 기술(Tech Stack)

### 1. 소프트웨어

- **언어:** C#
- **프레임워크:** .NET WinForms
- **주요 라이브러리:** `System.Windows.Forms`, `System.Text.RegularExpressions`
- **IDE:** Visual Studio 2022

### 2. 시스템 구조

- **메인 컨트롤러:** `Form1.cs` (주문 통합 관리 및 결제 로직)
- **메뉴 컴포넌트:**
    - `Form2Burger.cs`: 버거 메뉴 선택 및 개별 가격 계산
    - `Form2Drink.cs`: 음료 라디오 버튼 선택 로직
    - `Form2Side.cs`: 사이드 메뉴 체크박스 연동
- **UI 컴포넌트:** `UcBurger.cs`, `UsDrink.cs`, `UsSide.cs` (재사용 가능한 사용자 정의 컨트롤)

### 🛠 문제 해결 과정 (Troubleshooting)

### 1. 폼(Form) 간 데이터 전송 및 동기화 문제

- **문제점:** 메뉴 선택용 서브 폼(`Form2Burger` 등)에서 선택한 메뉴 이름과 가격 데이터를 메인 주문 폼(`Form1`)의 장바구니 리스트에 실시간으로 반영하는 데 어려움이 있었습니다.
- **해결 방안:** 각 서브 폼을 띄울 때 `Owner` 속성을 메인 폼으로 지정하고, `Form1`에 데이터를 수신하여 리스트박스를 업데이트하는 공개 메서드(`AddToOrder`)를 작성했습니다. 이를 통해 객체 지향적인 방식으로 폼 간 데이터를 안전하게 전달했습니다.

### 2. 문자열 파싱을 통한 실시간 금액 계산

- **문제점:** 장바구니(`ListBox`)에 저장된 "빅맥 (5,000원)"과 같은 혼합 문자열에서 숫자 부분만 추출하여 합계를 다시 계산하거나 항목 삭제 시 차감하는 로직 구현이 복잡했습니다.
- **해결 방안:** `System.Text.RegularExpressions.Regex`를 도입하여 문자열 내에서 숫자만 골라내는 정규표현식을 적용했습니다. 추출된 문자열을 `int.TryParse`로 변환하여 계산 로직의 안정성을 높이고 실시간 합계 금액 업데이트 기능을 구현했습니다.

### 3. 결제 프로세스의 유효성 검사

- **문제점:** 사용자가 메뉴를 하나도 고르지 않았거나 결제 수단을 선택하지 않은 상태에서 결제 버튼을 눌렀을 때 시스템 오류가 발생하거나 빈 주문이 들어가는 문제가 있었습니다.
- **해결 방안:** 결제 버튼(`btnPay`) 클릭 이벤트 내에 조건문을 추가하여 `lbCart.Items.Count`가 0이거나 결제 정보(`cbPay`)가 비어있을 경우 `MessageBox`를 통해 사용자에게 안내하고 프로세스를 중단시키는 유효성 검사 로직을 강화했습니다.

### 4. 코드 재사용성을 위한 사용자 정의 컨트롤 활용

- **문제점:** 버거, 사이드, 음료 등 메뉴의 종류가 늘어날 때마다 비슷한 UI와 코드를 반복해서 작성해야 하는 비효율이 발생했습니다.
- **해결 방안:** 메뉴의 이미지와 이름, 가격을 표시하는 공통 레이아웃을 `UserControl`(`UcBurger`, `UsDrink` 등)로 분리하여 컴포넌트화했습니다. 이를 통해 UI 수정 시 유지보수 시간을 단축하고 코드 가독성을 높였습니다.
